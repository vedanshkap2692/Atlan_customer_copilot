### RetriesÂ¶
(source: https://developer.atlan.com/sdks/python/#logging)

The SDK handles automatically retrying your requests when it detects certain problems:

When there is a403response indicating that permission for an operation is not (yet) available.

403

When there is a429response indicating that the request rate limit has been exceeded, and you need to retry after some time.

429

When there is a50xresponse indicating that something went wrong on the server side.

50x

If any request encounters one of these problems, it will be retried. Before each retry, the SDK will apply a delay using an exponential backoff.

(Currently the values for the exponential backoff are not configurable.)

For each request that encounters any of these problems, only up to a maximum number of retries will be attempted. (This is set to5by default.)

5

When to use async

Async is most beneficial when you have:

Multiple independent operationsthat can run concurrently

I/O-heavy workloadslike API calls, database queries, or file operations

Long-running operationswhere parallelization provides significant time savings

For simple, single operations, the synchronous client may be more straightforward to use.
